Command Settings

When enabled, you can send the following “/” commands directly in chat to control the bot.

Available Commands

  • /restart or /re: Restart the program (saves necessary data before restarting).
  • /summarize or /ms: Immediately summarize temporary memory into memory fragments (regardless of whether memory is enabled).
  • /disable_proactive or /da: Disable proactive messaging.
  • /enable_proactive or /ea: Enable proactive messaging.
  • /clear_memory or /cl: Clear the short-term context and temporary memory for the current chat window. Use when AI replies are abnormal (e.g., garbled text or chain-of-thought).
  • /enable_voice or /ev: Allow voice calls for scheduled reminders (private chat only).
  • /disable_voice or /dv: Disable voice calls for scheduled reminders.

Scheduled Bot Restart Config

Auto-restart feature is disabled. Settings below have no effect.
This is an experimental feature. When enabled, the program may auto-restart under specific conditions to resolve performance degradation and slow responses from long runtime. Chat context and reminder list are automatically saved before restart.
When enabled, the program will attempt auto-restart based on the conditions set below.
After the program runs continuously for this duration, it will begin checking if the inactivity restart condition is met. E.g., 2.0 means check after 2 hours of runtime.
After the program reaches the uptime threshold, if there are no messages received or bot activity for this many continuous minutes, a restart will be executed. E.g., 15 means 15 continuous minutes of inactivity required.
Important Notes:
  • Scheduled restart will interrupt all ongoing processing, including unfinished conversations and message queues.
  • Settings changed via the Config Editor require manual restart to take effect; scheduled restart will not apply these changes.
  • Scheduled restart automatically avoids scheduled reminders to prevent missing them during restart.
